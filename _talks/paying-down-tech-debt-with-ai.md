AI can help teams ship features faster - often by accepting a bit of technical debt along the way. But the same tools can also help repay that debt. Over the past year, we used AI to reduce compile-time cyclic dependencies, eliminate test flakes, and improve code standards. While AI-generated code still introduces some issues, it can also become a powerful tool for improving long-term code quality.


**Key takeaways:**

* Practical ideas for using AI to identify and reduce technical debt
* Techniques for improving code quality with AI-assisted tooling
* Approaches to automating code quality improvements within your development workflow

**Target audience:**

* Anyone who builds software.
